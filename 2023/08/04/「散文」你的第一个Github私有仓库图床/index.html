前言

拥有一个免费图床一直是我的众多执念之一。
市面上不乏有很多存储服务商，比如阿里云、腾讯云、七牛云、SM.MS、路过图床等等，但或多或少都有订阅服务，诸如免费会员，一档会员，二档会员，总之就是用不爽。此前我的图一直托管在腾讯云 OSS 桶里面，时不时扣我几毛钱，敢吸我的血？我 tm 把你停了！
Github —— 私有库 + 存储无上限 = 完美图床啊 😍（虽然有时候速度有点慢），开搞开搞 🚼。

思路

域名

⚠️ 如果域名是从国内服务商购买（腾讯云、阿里云），你需要依法按照工信部规定完成备案后才能进行后续配置，且域名不能单独备案，你还需要搞一个 IP 地址一起绑定。

首先你需要购入一个域名，你可以从诸多域名商去选择：

  1. 腾讯云
  2. 阿里云
  3. NameSilo
  4. NameCheap
  5. Google Domains
  6. Godaddy
  7. …

购入域名后，需要去解析域名的 DNS ，不然是没法正常用的，大概理解为生了小孩得给小孩上户口，不然就是黑户啥都干不了。
一般域名商会给你配置的教程，有的域名商默认会把 DNS 设置为它自己的。配置完成后，在你本地启动终端，执行ping <your-domain>，如果 ping 通了就说明成功了。

私有库

Github 2020 年就宣布向所有用户和团队提供不限制协作人数的私有仓库。GitHub，启动！

第一步，创建一个私有仓库：

  1. 填仓库的名字
  2. 填仓库介绍（可选）
  3. 设置仓库类型为 Private
  4. 添加一个 README.md，选这个默认会创建一个 master 分支，省的你本地再建一个仓库再去关联
  5. create repo


创建好，咱们手动往仓库里传一张图片，上传完成后大概长下面这样。

然后对图片右键，新标签页打开图片，复制图片地址，粘贴到随便哪个 MD 文档中看看，是可以展示的，这为 Github 作为图床垫定了可行性的基础。
我们再来看这个地址，重点关注最后一个尖括号<occasional-token>，这个 token 是一个临时性的令牌，几分钟就会失效，因此直接复制这个地址是不可行的，这就轮到后面 Cloudflare 登场解决了。

https://raw.githubusercontent.com/<github-username>/<github-repo-name>/<github-repo-branch>/github-private-pic-bed-202307211632485.png?token=<occasional-token>

第二步，仓库创建好了，我们还需要生成一个令牌给 Cloudflare 和 PicGo 使用，当他们的请求到 Github 的时候直接拎包入住。
点右上角头像，拉起 Github 右侧边栏，点进 Settings。

然后滑到底，点 Developer settings，再点到底选 Tokens，右侧点生成一个新 Token。


再然后随便起个名字，选过期时间，只勾选 repo 的权限即可，因为我们只用得到这些。

最终会拿到一个 Token 字符串，这个需要提前复制一下，不然退出去就看不到了。

Github 这部分到此结束，嘻嘻。

Cloudflare

Cloudflare 是一家提供 CDN、DDoS 防护、DNS 服务等的云服务商。它通过全球部署的服务器网络,帮助客户提高网站和应用的性能和安全性。
文章一开始提到了使用 Cloudflare 代理请求去拿图片，这里主要用的是 Cloudflare Worker 服务，它是 Cloudflare 提供的 serverless 服务。它可以让用户在 Cloudflare 全球边缘网络上部署代码,用来自定义网络请求处理逻辑。比如修改响应头、数据加密、重写 URL 等。

遁入 Cloudflare 门下

首选需要将自己域名的 DNS 服务器变更为 Cloudflare 的，如果不操作则没法用他的代理、加速等服务，总之这是一个基础且必要步骤。
在这个页面点击添加站点，然后输入主域名点继续。


这时候 Cloudflare 会让你选氪金服务，穷鬼人，无脑选免费，继续！

下面 Cloudflare 会扫描的 DNS 的记录，然后我们按提示前往域名商的控制台设置 DNS 服务器，过程就省略了。

上面设置完等几分钟配置生效，看见下方的面板就说明咱们成功遁入 Cloudflare 门下。

Worker，启动！

页面选创建 Worker，一路来到这个页面，名称随便起，代码见下面。

/**

* Welcome to Cloudflare Workers! This is your first worker.

*

* - Run "npm run dev" in your terminal to start a development server

* - Open a browser tab at http://localhost:8787/ to see your worker in action

* - Run "npm run deploy" to publish your worker

*

* Learn more at https://developers.cloudflare.com/workers/

*/

// Website you intended to retrieve for users.

const upstream = "raw.githubusercontent.com";

// Custom pathname for the upstream website.

// (1) 填写代理的路径，格式为 /<用户>/<仓库名>/<分支>

const upstream_path = "";

// github personal access token.

// (2) 填写github令牌

const github_token = "";

// Website you intended to retrieve for users using mobile devices.

const upstream_mobile = upstream;

// Countries and regions where you wish to suspend your service.

const blocked_region = [];

// IP addresses which you wish to block from using your service.

const blocked_ip_address = ["0.0.0.0", "127.0.0.1"];

// Whether to use HTTPS protocol for upstream address.

const https = true;

// Whether to disable cache.

const disable_cache = false;

// Replace texts.

const replace_dict = {
  $upstream: "$custom_domain",
};

addEventListener("fetch", (event) => {
  event.respondWith(fetchAndApply(event.request));
});

async function fetchAndApply(request) {
  const region = request.headers.get("cf-ipcountry")?.toUpperCase();

  const ip_address = request.headers.get("cf-connecting-ip");

  const user_agent = request.headers.get("user-agent");

  let response = null;

  let url = new URL(request.url);

  let url_hostname = url.hostname;

  if (https == true) {
    url.protocol = "https:";
  } else {
    url.protocol = "http:";
  }

  if (await device_status(user_agent)) {
    var upstream_domain = upstream;
  } else {
    var upstream_domain = upstream_mobile;
  }

  url.host = upstream_domain;

  if (url.pathname == "/") {
    url.pathname = upstream_path;
  } else {
    url.pathname = upstream_path + url.pathname;
  }

  if (blocked_region.includes(region)) {
    response = new Response(
      "Access denied: WorkersProxy is not available in your region yet.",

      {
        status: 403,
      }
    );
  } else if (blocked_ip_address.includes(ip_address)) {
    response = new Response(
      "Access denied: Your IP address is blocked by WorkersProxy.",

      {
        status: 403,
      }
    );
  } else {
    let method = request.method;

    let request_headers = request.headers;

    let new_request_headers = new Headers(request_headers);

    new_request_headers.set("Host", upstream_domain);

    new_request_headers.set("Referer", url.protocol + "//" + url_hostname);

    new_request_headers.set("Authorization", "token " + github_token);

    let original_response = await fetch(url.href, {
      method: method,

      headers: new_request_headers,

      body: request.body,
    });

    let connection_upgrade = new_request_headers.get("Upgrade");

    if (connection_upgrade && connection_upgrade.toLowerCase() == "websocket") {
      return original_response;
    }

    let original_response_clone = original_response.clone();

    let original_text = null;

    let response_headers = original_response.headers;

    let new_response_headers = new Headers(response_headers);

    let status = original_response.status;

    if (disable_cache) {
      new_response_headers.set("Cache-Control", "no-store");
    } else {
      new_response_headers.set("Cache-Control", "max-age=43200000");
    }

    new_response_headers.set("access-control-allow-origin", "*");

    new_response_headers.set("access-control-allow-credentials", "true");

    new_response_headers.delete("content-security-policy");

    new_response_headers.delete("content-security-policy-report-only");

    new_response_headers.delete("clear-site-data");

    if (new_response_headers.get("x-pjax-url")) {
      new_response_headers.set(
        "x-pjax-url",

        response_headers

          .get("x-pjax-url")

          .replace("//" + upstream_domain, "//" + url_hostname)
      );
    }

    const content_type = new_response_headers.get("content-type");

    if (
      content_type != null &&
      content_type.includes("text/html") &&
      content_type.includes("UTF-8")
    ) {
      original_text = await replace_response_text(
        original_response_clone,

        upstream_domain,

        url_hostname
      );
    } else {
      original_text = original_response_clone.body;
    }

    response = new Response(original_text, {
      status,

      headers: new_response_headers,
    });
  }

  return response;
}

async function replace_response_text(response, upstream_domain, host_name) {
  let text = await response.text();

  var i, j;

  for (i in replace_dict) {
    j = replace_dict[i];

    if (i == "$upstream") {
      i = upstream_domain;
    } else if (i == "$custom_domain") {
      i = host_name;
    }

    if (j == "$upstream") {
      j = upstream_domain;
    } else if (j == "$custom_domain") {
      j = host_name;
    }

    let re = new RegExp(i, "g");

    text = text.replace(re, j);
  }

  return text;
}

async function device_status(user_agent_info) {
  var agents = [
    "Android",

    "iPhone",

    "SymbianOS",

    "Windows Phone",

    "iPad",

    "iPod",
  ];

  var flag = true;

  for (var v = 0; v < agents.length; v++) {
    if (user_agent_info.indexOf(agents[v]) > 0) {
      flag = false;

      break;
    }
  }

  return flag;
}

点部署，然后进入 Worker 的配置页，选触发器，这时候配上买的域名即可，如果你没遁入 Cloudflare 门下，这步就会失败。说个题外话，如果你的主域名有其他用处，可以从刚刚域名的面板进入 DNS 配置，切一个子域名给图床用，过程嘛不演示了。

PicGo

PicGo 是一个开源的图床上传工具,支持多平台使用。它可以一键上传图片并获取图片 URL,支持主流图床如 GitHub、腾讯云、又拍云等。PicGo 简单易用,不需要繁琐配置就可以快速上传 markdown、Word 文档中的图片。同时支持拖拽图片、快捷键上传、自动复制 URL 等功能。

跟着提示来配置，教程基本上要结束喽。

还有些配置在 PicGo 设置这儿，如自动时间戳命名、上传后自动复制 URL 还是挺方便的。
