前言

TAG： #字符串加强 #Unicode #遍历字符串 #模版字符串 #编译模版 #标签模版

ES6 中针对 string 拓展了一些很好用的功能，主要分为两大类：

  1. 加强对Unicode的支持
  2. 没用过就不是前端的模版字符串

我对 Unicode 的了解很少，几乎没在开发中碰见过处理 Unicode 的场景，详细内容嘛待我 Unicode 学成归来，再作分解 👀。

string 的遍历

ES6 中为 string 部署了 Iterator 接口，你可以像玩数组一样的去玩字符串～

const str = "abcdefg";
const strArr = [...str];
console.log(strArr); // ['a', 'b', 'c', 'd', 'e', 'f', 'g']

str.forEach((word) => {
  console.log(word); // 'a' 'b' 'c' 'd' 'e' 'f' 'g'
});

模板字符串

ES6 之前的模版字符串，各种拼接，JQ 传巨多参数。实现过程可谓山路十八弯，劳累啊 😮‍💨。

ES6 带来的模版字符串直接一句话搞定是高级版的字符串，可以用来定义多行字符串，可以在中间穿插各种变量和表达式，也可以当作普通字符串使用。这些特性无不印证了一句话——梭哈多是一种智慧啊～。

基础

基础使用十分的简单，用一对反括号将你的内容框住，然后你就可以开始自由发挥了。上面还漏提了一点，它甚至能嵌套使用，回想起以前你单引号、双引号来回切换的痛苦了吗？

const name = "张三";
const age = 99;
const ddl = {
  age: 100,
};
const getRes = () => !!(ddl.age - age) && "活";

console.log(`${name}今年${age}岁
还有${ddl.age - age}年可以${getRes()`在人间`}`);

张三今年99岁;
还有1年可以活在人间;

高级 1-编译模版

将下面的代码复制到浏览器打开～，你就会看见一个大大的窝嫩叠😆

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <div id="compileDom"></div>
    <script>
      // 创建一个compile函数
      const compile = (content) => {
        // 渲染Dom
        return `<h1>${content}</h1>`;
      };
      // 获取到渲染容器
      const dom = document.getElementById("compileDom");
      // 赋值
      dom.innerHTML = compile("窝嫩叠");
    </script>
  </body>
</html>

高级 2-标签模版

在一个函数后面跟着写上模板字符串，那么这个函数将被用于处理这个模板，这种写法被称为标签模版。标签模版本质上并不是模版，而是函数调用的一种特殊形式。

const tag = (strings, ...values) => {
  console.log(strings, values);
};

tag`张三叫 ${"张三丰"} 是 ${"铜锣湾"} 扛把子`;

// [ '张三叫 ', ' 是 ', ' 扛把子' ] [ '张三丰', '铜锣湾' ]

可以发现使用标签模版的方式，原模版非填充的内容回被拆解成一个string[],而填充的值会作为后续参数传入，可以通过拓展运算符来全部获取，完整来看其实 tag 执行时的传参是这样的tag([ '张三叫 ', ' 是 ', ' 扛把子' ], '张三丰', '铜锣湾')。

下面给大家整个活儿！使用标签模版还原模板字符串 🐒

const tag = (strings, ...values) => {
  let originString = "";
  values.forEach((item, idx) => {
    originString += strings[idx] + item;
  });
  return originString + strings[values.length];
};

const originString = tag`张三叫 ${"张三丰"} 是 ${"铜锣湾"} 扛把子`;

console.log(originString);
// 张三叫 张三丰 是 铜锣湾 扛把子
console.log(`张三叫 ${"张三丰"} 是 ${"铜锣湾"} 扛把子`);
// 张三叫 张三丰 是 铜锣湾 扛把子

标签模版应用场景

过滤 HTML 字符串，转义恶意内容：将下面的代码放入浏览器后，通过开发者工具查看元素可以发现 safeDom 下面就是一个字符串，dangerousDom 下面却带有一个<script></script>标签，这就有可能被不怀好意的人用来跑恶意代码 😱😱。

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <div id="safe"></div>
    <div id="dangerous"></div>
    <script>
      const safeFilter = (template, ...values) => {
        const [dangerousCTX] = values;
        const safeCTX = dangerousCTX
          .replace("<", "&nbps;")
          .replace("&", "&nbps;")
          .replace(">", "&nbps;");
        return template[0] + safeCTX + template[1];
      };

      const dangerous = "<script defer>alert(我要盗号！！！);<\/script>";
      const safe = safeFilter`<p>${dangerous} may be dangerous</p>`;

      const safeDom = document.getElementById("safe");
      const dangerousDom = document.getElementById("dangerous");

      safeDom.innerHTML = safe;
      dangerousDom.innerHTML = `<p>${dangerous} may be dangerous</p>`;
    </script>
  </body>
</html>

国际化处理：写了一个比较简陋的的国际化方法 🤡，大体就是将除填充部分进行国际化处理～

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <div id="ml"></div>
    <div id="nl"></div>
    <script src="https://code.jquery.com/jquery-3.6.0.min.js"></script>
    <script>
      const dictionary = {
        Hello: "你好",
        World: "世界",
        你好: "Hello",
        世界: "World",
      };

      const translate = (str) => {
        const strArr = str.split(" ");
        const translate_strArr = (Array.isArray(strArr) ? strArr : []).map(
          (str) => {
            return dictionary[str] || str;
          }
        );
        console.log(str, strArr, translate_strArr);
        return translate_strArr.join(" ");
      };

      const generateString = (strings, ...values) => {
        let originString = "";
        values.forEach((item, idx) => {
          originString += translate(strings[idx]) + item;
        });
        return originString + translate(strings[values.length]);
      };

      $("#ml").html(generateString`Hello World ! ${"张三"}!`);
      $("#nl").html(generateString`你好 世界 ! ${"张三"}!`);
    </script>
  </body>
</html>

解释器：善用模板字符串，你甚至能在用原生 JavaScript 跑 Java 的代码，这俩货不能说有一点关系，只能说一点关系也没有。下面也是一个非常简陋的的版本，而且和标签模版也没多大关系 🤣，想要真正实现还得钻研一波 AST。

const JavaCode = `public class HelloWorld {
  public static void main(String []args) { 
    System.out.println("Hello World!");
  }
}`;

const interpreter = (JavaCode) => {
  const str_origin_arr = JavaCode.split(" ");
  const str_effect_arr = str_origin_arr.filter((str) => !!str);

  const funcInfos = {};

  str_effect_arr.forEach((item, idx) => {
    if (item === "class") {
      funcInfos.functionName = str_effect_arr[idx + 1];
    }
    if (item.includes("main")) {
      funcInfos.main = () => {};
    }
    if (item.includes("System.out.println")) {
      const wholeString = `${str_effect_arr[idx]} ${str_effect_arr[idx + 1]}`;
      const content = wholeString.match(/\([^\)]+\)/g)[0];
      const res = content.substring(2, content.length - 2);
      funcInfos.main = () => {
        console.log(res);
      };
    }
  });

  funcInfos.main();
};

interpreter(JavaCode);

模版字符串的限制

前面提到过模板字符串中可以嵌套字符串、表达式等等内容，但处理一些对于 JS 来说是转义概念的内容就力不从心了，例子见参考的那章（因为对原文中提到的 LaTEZ 这个东西完全没有了解 😅）。

新增的方法

用于处理 Unicode 的方法会放在这个头部里，埋个坑，究其原因还是怪我太菜了 😭

string.includes()与 string.startWith()与 string.endWith()

ES6 终于更新查找字符串的方法，indexOf 不在孤独。

  • string.includes(ctx: string, startPosition: number)：返回一个布尔值，表示是否找到目标字符串。
  • string.startWith(ctx: string, startPosition: number)：返回一个布尔值，表示是否在原字符串头部找到目标字符串。
  • string.endWith(ctx: string, startPosition: number)：返回一个布尔值，表示是否在原字符串尾部找到目标字符串。
const test_str = "zhangsan is a man";

console.log(test_str.includes("man", 8)); // true
console.log(test_str.startWith("zhangsan", 8)); // false
console.log(test_str.endWith("zhangsan", 8)); // true

string.repeat()

repeat 方法会返回一个新的字符串，接受一个 number 类型的参数，代表重复几次。

const test_str = "shit";

console.log(test_str.repeat(2)); // 'shitshit'
console.log(test_str.repeat(0)); // ''
console.log(test_str.repeat(2.4)); // 取舍退位，'shitshit'
console.log(test_str.repeat(Infinity)); // ERROR
console.log(test_str.repeat(-1)); // ERROR
console.log(test_str.repeat(-0.9)); // 取整进位，''
console.log(test_str.repeat("shit")); // 尝试转化为number类型，''
console.log(test_str.repeat("1")); // 尝试转化为number类型，'shit'
console.log(test_str.repeat("NaN")); // 转化为0，''

string.padStart()与 string.padEnd()

字符串补全方法，接受两个参数，第一个参数为补全长度，第二个参数为补全内容，如果省略第二个参数默认用空格补齐。

const str_start = "father";
const str_end = "son";

console.log(str_start.padStart(16, "I am your ")); // I am your father
console.log(str_end.padEnd(20, ",I am your father")); // son, I am your father
console.log(str_start.padStart(10)); //     father

string.trimStart()与 string.trimEnd()

消除字符串头部、尾部的空格，行为和 string.trim()一致，且他们都是返回新的字符串，不会修改原字符串。

const str_start = "father";
console.log(str_start.padStart(10).trimStart()); // father

string.matchAll()与 string.replaceAll()

这两兄弟会放在正则那章提到，大致意思就是matchAll会去匹配字符串中所有符合的匹配项，而replaceAll不同于以往的 replace 只能替换一项，它能替换所有匹配项的内容！

string.at()

at()接受一个 number 类型的参数。ES6 给 string 部署了 iterator 接口，这里就是返回字符串中索引值对于的内容，且它还支持倒索引。但是这个东西还在提案中，并没广泛支持 😅。

const test_str = "test";

console.log(
  test_str.at(0), // t
  test_str.at(1), // e
  test_str.at(2), // s
  test_str.at(3), // t
  test_str.at(4), // undefined
  test_str.at(-1) // t（最后一个）
);
