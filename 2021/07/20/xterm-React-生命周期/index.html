React父子生命周期执行顺序

这里就直接抛出结论，后面会系统性的梳理一下React的生命周期

  • 当父组件中状态发生变化（包括子组件的挂载以及卸载）时，会触发自身对应的生命周期以及子组件的更新，且要分为两种情况

    • render 以及 render 之前的生命周期，则 父组件先执行
    • render 以及 render之后的声明周期，则子组件先执行，并且是与父组件交替执行

  • 当子组件自身状态改变时，不会对父组件产生副作用的情况下，父组件不会进行更新，即不会触发父组件的生命周期勾子。

  • 当子组件进行卸载时，只会执行自身的 componentWillUnmount 生命周期，不会再触发别的生命周期

什么是xterm？

xterm官网

安装： npm install xterm –save / yarn add xterm

xterm的官网是这么介绍它的————xterm.js是一款用TypeScript编写的前端组件，它可以让应用程序在浏览器中给用户带来完整的终端功能。

这里用官方提供的最佳示例来讲一下。首先需要引入xterm库和它的样式。xterm会暴露一个构造函数Terminal给我们，我们可以用其来创建一个Terminal实例。然后我们需要一个div来作为Terminal的容器让它能在浏览器上展示出来。

<!doctype html>
  <html>
    <head>
      
      <!-- 引用xterm的样式文件，不然会 -->
      <link rel="stylesheet" href="node_modules/xterm/css/xterm.css" />
      
      <!-- 引用xterm库 -->
      <script src="node_modules/xterm/lib/xterm.js"></script>
      
    </head>
    <body>
      
      <!-- 设置DOM容器 -->
      <div id="terminal"></div>
      
      <script>
        
        // 创建terminal实例
        var term = new Terminal();
        
        // 将实例和DOM绑定起来
        term.open(document.getElementById('terminal'));
        
        // 在Termianl中写入Hello World
        term.write('Hello World');
      </script>
    </body>
  </html>

开发的心路历程

环境： React，React Hooks，Antd，xterm

需求： 新增一个按钮，点击后弹出一个Web-Shell模态框

拿到需求首先想到这个组件会涉及多个页面，直接提成一个公共组件。控制Modal框的显隐布尔值和方法是由外部的State管理作为参数传入的，同时也支持外部控制Modal的title显示内容。

// 引入依赖
import React from 'react';
import { Modal } from 'antd';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

// 创建函数组件
const Terminal = ({ visible, setVisible, name }) => {
  
  return (
      <Modal
      title={`终端-${name}`}
        visible={visible}
      onClose={() => { setVisible(false) }}
    >
      <div id="terminal"></div>
    <Modal />
  );
};

// 导出组件
export default Terminal;

然后我们需要组件挂载阶段去把Terminal和DOM绑定起来同时通过WebSocket和后端建立连接。函数组件怎么去实现componentDidMount呢？那当然无敌的useEffect啦！new Terminal中填的参数是一些样式参数，具体和更多参数可以去官网查。为了防止内存泄漏，我们还需要给Modal加上destroyOnClose这个属性，以及给useEffect加上return方法。

import React, { useEffect } from 'react';
import { Modal } from 'antd';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

const Terminal = ({ visible, setVisible, name }) => {
  
  useEffect(() => {
    
    // 创建socket实例
    const socket = new WebSocket('ws://yoururl.com/.....');
    
    // 创建terminal实例
    const terminal = new Terminal({
      cursorStyle: 'underline', // 光标样式
      cursorBlink: true, // 光标闪烁
      theme: {
        foreground: '#dddddd', // 字体
        cursor: 'gray', // 设置光标
      },
      windowsMode: true, // 自动换行
    });
    
    // 获取DOM实例
    const terminalDOM = document.getElementById('terminal');
    
    // 绑定DOM和Terminal实例
    terminal.open(terminalDOM);
    
    // 监听socket的开始和结束
    // 开始
    socket.onopen = () => {
      terminal.write('Welcome to Terminal \r\n$');
    };
    // 结束
    socket.onclose = () => {
      terminal.write('Bey,Bey \r\n$');
    };
    
    // 当组件被卸载的时候断开socket链接
    return () => {
      socket.close();
    };
  }, []);
  
  return (
      <Modal
      title={`终端-${name}`}
        visible={visible}
            // 当Modal关闭的时候销毁整个组件
      destroyOnClose
      onClose={() => { setVisible(false) }}
    >
      <div id="terminal"></div>
    <Modal />
  );
};

export default Terminal;

做完上面的步骤我们来看一下效果，结局非常令人舒适，DOM获取不到。犯了低级错误，错误判断了父组件和子组件的生命周期的执行顺序。于是把div提出来单独作为一个TerminalDetail组件，同时把相关操作迁移过去。

import React, { useEffect } from 'react';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

const TerminalDetail = () => {
  useEffect(() => {
    
    const socket = new WebSocket('ws://yoururl.com/.....');
    const terminal = new Terminal({
      cursorStyle: 'underline',
      cursorBlink: true,
      theme: {
        foreground: '#dddddd',
        cursor: 'gray',
      },
      windowsMode: true,
    });
    
    const terminalDOM = document.getElementById('terminal');
    
    terminal.open(terminalDOM);
    
    socket.onopen = () => {
      terminal.write('Welcome to Terminal \r\n$');
    };
    
    socket.onclose = () => {
      terminal.write('Bey,Bey \r\n$');
    };
    
    return () => {
      socket.close();
    };
  }, []);
  
  
  return <div id="terminal"></div>
};

export default TerminalDetail;

Terminal组件也需要做相应的改造。这时切换到浏览器，Terminal显示正常了🎉🎉🎉。

import React from 'react';
import { Modal } from 'antd';
import TerminalDetail from './TerminalDetail';

const Terminal = ({ visible, setVisible, name }) => {
  
  return (
      <Modal
      title={`终端-${name}`}
        visible={visible}
            // 当Modal关闭的时候销毁整个组件
      destroyOnClose
      onClose={() => { setVisible(false) }}
    >
      <TerminalDetail />
    <Modal />
  );
};

export default Terminal;

现在我们来给TerminalDetail添加功能，也很简单👌。在浏览器的Terminal中输入的时候我们能通过terminal.onData方法获取每次输入的内容（⚠Terminal绑定DOM后才能设置哈），我们把这些东西通过socket传给后端。后端又把真实终端中的内容返回给前端展示，前端用socket.onmessage来接收返回的内容（⚠WebSocket成功连接后哈）。去浏览器里体验一把，输入echo Hello World，显示正常🎉🎉🎉。

import React, { useEffect } from 'react';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

const TerminalDetail = () => {
  useEffect(() => {
    
        // 省略......
    
    // 前端和后端交互
    terminal.onData((e) => {
      socket.send(e);
    });
    // 后端和前端交互
    socket.onmessage = (e) => {
      terminal.write(e?.data);
    };
    
    // 省略......
  }, []);
  
  
  return <div id="terminal"></div>
};

export default TerminalDetail;

这里还有便捷的插件————xterm-addon-attach，这个插件能帮我们自动和socket交互，省的我们自己去监听事件。具体使用就是在Terminal实例生成后，绑定DOM前，构造插件的实例，然后用loadAddon这个方法去加载插件就行了。

import React, { useEffect } from 'react';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

// 引入插件
import { AttachAddon } from 'xterm-addon-attach';

import React, { useEffect } from 'react';
import { Terminal } from 'xterm';
import 'xterm/css/xterm.css';

const TerminalDetail = () => {
  useEffect(() => {
    
        // 省略......
    
    // 创建插件实例，绑定我们的socket
    const attachAddon = new AttachAddon(socket);
    // 加载插件
    terminal.loadAddon(attachAddon);
    
    // 省略......
  }, []);
  
  return <div id="terminal"></div>
};

export default TerminalDetail;

总结

xterm踩了不少坑，比如onData输入是没问题但是像退格之类的操作符就会报错，原因是真实终端的数据（UTF-16/UCS-2）由后端是原封不动地返回给前端（UTF-8 ），进制对不齐就出问题了。还有最开始调研xterm工作做的不足，AttachAddon插件和onData同时使用，这就导致程序让socket发送了两遍内容，后端也是直接返回两次内容，前端输入a，终端上直接显示aa。文中开发过程中生命周期的问题虽然一笔带过，其实我还想了挺久🙇‍♂️，希望以后不要再犯这样的低级错误。
