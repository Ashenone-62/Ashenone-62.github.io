小记 Mac 配置前端环境

前言

我的老 Win 本已经用了快接近四年的时间，前段时间电池光荣牺牲了，加上在公司尝到 Mac 的香，于是一咬牙一跺脚换了台 m1 的 MacBook Air😖 钱包君直接当场去世。

新电脑一到，首当其冲要做的便是配置开发环境。从 Win 转到 Mac 很多操作也要跟着变，不过咱作为开发可不能嫌麻烦啊 😊 生产力，开冲！

软件清单

  • git（MacBook 原生提供）
  • node，npm（安装时需要 Rosetta，安装完成后原生支持）
  • yarn（包管理工具，通过官网提供 curl 命令行进行安装）
  • chrome（直接去官网下，原生支持）
  • vscode，webstorm（前端 ide，不过后者似乎有段时间没更新了）
  • nodemon（一个 node 库，可以用来热更新 node 服务）
  • 还有一些其他乱七八糟的库…

配置环境变量

安装 yarn 的时候，yarn 是被成功安装了，然后执行的时候却报找不到 yarn module 的错误。开发是敏锐的 🤔，这一看就是环境变量配置的问题，yarn 的环境变量是配在.zshrc这个文件下的，使用 open 或者 vim 命令打开我们 home 目录下的文件。

open ~/.zshrc
vim ~/.zshrc

打开后发现 yarn 的 PATH 是配置好了的，但是为什么没有生效呢？参考 Linux 是 MacOS 的爸爸这一事实有段联想到是配置没有更新导致的没有生效。于是乎就得请出.bash_profile这个文件，这个文件是干啥的勒？它是能让每个用户都可使用该文件输入专用于自己使用的 shell 信息,当用户登录时,该文件仅仅执行一次!默认情况下,他设置一些环境变量,执行用户的.zshrc 文件。如果你有.bash_profile这个文件那就很好办，在.zshrc这个文件的尾巴上写入下面的命令行就 👌 了。

source ~/.bash_profile

如果你没有这个文件，那你就得在 home 目录手动创建了（没错我就没有 😓）。手动创建好,然后重复上述动作再尾巴上加一行代码就 👌 了，接下来输入一下yarn -v就能看到版本信息了。

以为就这完了？😁 想的美，现在你还得给接下来安装 npm 全局包留环境变量的口子。在.bash_profile中输入下面的命令。

export PATH=~/.npm-global/bin:${PATH}

最后将 npm 全局安装的包的位置重定向到 npm-global 这个文件夹下就竣工了。

npm config set prefix '~/.npm-global'
yarn config set prefix '~/.npm-global'

彻底摆脱not found xxx Module的烦恼 😆。

Mac 的 24 小时使用报告

真香！！

  • webpack 打包速度，原来两分钟起步，现在直接十几秒的事！
  • yarn 安装依赖，原来三分钟起步，现在直接瞬间完成！
  • 触摸板顺滑程度完全代替掉鼠标！
  • 使用连续使用 11 个小时，还剩 23%的电！离了电源撑不住 5 分钟的神船看了直接泪流满面！
