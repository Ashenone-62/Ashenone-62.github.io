前言

在智能移动设备刚开始发展的时候，人们访问 Web 网页的时候只能看到占满视窗大小的 PC 页面，可以通过缩放来浏览。如今移动设备在日程生活的比重已然到 TOP1 , Web 网页针对移动设备的体验也有了质的飞跃。在布局结构不那么复杂的网站通常会使用响应式布局（ Responsive Layout ）；而较为复杂的页面，会选择做双栈，通过JS来获取设备信息来进行跳转。

meta标签

我们会在 HTML 的头标签中经常看到下面代码块中的标签

  • name
    • viewport –> 在移动设备上设置原始大小显示和是否缩放的声明
  • viewport
    • width = device-width –> 视口宽度等于设备宽度(1 : 1)
    • initial-scale = 1.0 –> 初始缩放倍数
    • maximum-scale = 1.0 –> 最大缩放倍数
    • user-scalable = 0 –> 是否允许用户进行缩放
<meta name="viewport" content="width=device-width; initial-scale=1.0; maximum-scale=1.0; user-scalable=0;" />

em 与 rem

em 与 rem是 CSS 的长度单位。

em 会不断向上查找，直到找到设置了font-size大小的那一级，根据那一级的字体大小来计算像素值。

/* parent3的字体会根据parent1的字体来设置，在浏览器的检查模式下会显示 20px */
.parent1{
    font-size: 10px;
}

.parent1 .parent2{
    background: red;
}

.parent1 .parent2 .parent3{
    font-size: 2em;
}

rem直接以根元素的字体大小来计算像素值。

/* 	
    parent3的字体会直接根据根元素(html)的字体设置，在浏览器的检查模式下会显示 32px 
*/
html{
    /* 一般来说浏览器的默认大小就是16px */
    font-size: 16px;
}

.parent1{
    font-size: 10px;
}

.parent1 .parent2{
    background: red;
}

.parent1 .parent2 .parent3{
    font-size: 2rem;
}

媒体查询@media

在 CSS 中可以使用 @media 查询可以针对不同的媒体类型定义不同的样式。@media 可以针对不同的屏幕尺寸设置不同的样式，特别是如果你需要设置设计响应式的页面。当你重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。

/*
    写法
    @media (condition) {
        ...style
    }
*/

/* 当视口宽度大于等于 1200px 的时候，box类下的背景色为红色 */
@media (min-width: 1200px) {
    .box{
        backgroundColor: red
    }
}
/* 当视口宽度大于等于 768px 的时候，小于等于 1199px 的时候，box类下的背景色为蓝色 */
@media (max-width: 1199px) and (min-width: 768px) {
    .box{
        backgroundColor: blue
    }
}
/* 当视口宽度大于等于 400px 的时候，小于等于 767px 的时候，box类下的背景色为蓝色 */
@media (max-width: 767px) and (min-width: 400px) {
    .box{
        backgroundColor: yellow
    }
}

<!-- index.html -->

<link rel="stylesheet" media="screen and (max-width: 1199px)" href="css/mobile.css">

初始化根元素字体

根元素字体一般默认 16 px，不同浏览器的默认大小可能不同，我这儿用的是 chrome 。chrome 的最小字体就是 12 px，如果需要更小需要设置。这里设置为 62.5% 后就是 10 px, 10 的倍数更方便计算

/* common.css */

html {
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
    font-size: 62.5%;
}

清除浮动

有使用浮动的地方就要清除浮动。如果不清除可能就会导致内外边距显示错误；盒子不能正常撑开影响图片显示；边框不能正常撑开显示错误等等一些问题。

/* common.css */

.clearfix::after {
    content: "";
    display: block;
    clear: both;
}

优化列表类元素观感

这里主要围绕.course-board，.inner和.course-card来开发。

.course-board是120 rem 定宽（1200px）。先分别给.course-board（黑 border），.inner（红 border）加上不同颜色的 border 。这里有一个地方需要注意，那就是卡片边框，如果每次当鼠标移上去再加一个 border 就会导致 div 被撑开，所以所以我们先给一个 border ，将他的颜色设置为 transparent ，当鼠标移上去就修改它的 border 颜色。

设置.course-card宽度为30 rem，那么四个.course-card理应占满整个.course-board，但是由于浮动的设置第四个卡片元素的图片实际上并不能紧紧贴合黑色边框。将其宽度改为25%，四个卡片等分.course-board就能占满黑框了。这时候设置红框的作用就出来了，由于我们是提前给了卡片边框还有边距的，所以四个卡片实际宽度是大于120rem的，但是由于颜色是 transparent 所以我们能看见的就只有占满 120rem 的卡片，但是鼠标移上去边框和边距又能被感知到，观感直接 up 。

双栈

如果页面过于复杂，比如有着复杂分类和结构的电商，开发时候多半会选择双栈开发—— PC 端和移动端分开。通过 JS 判断用户的设备类型来跳转到对应的端。

/* 移动端 */
if(!(navigator.userAgent.match(/(iPhone|iPod|Android|ios)/i))){
    window.location.href = "http://xxx.com"
}

/* PC端 */
if(navigator.userAgent.match(/(iPhone|iPod|Android|ios)/i)){
    window.location.href = "http://m.xxx.com"
}

common.js

/* 创建自执行函数 */
;(function (doc) {
    
    /* 获取header的菜单按钮，与导航栏 */
    let MenuBtn = document.querySelector('.header .fa-bars'),
        Nav = document.querySelector('.header .header-nav')
    
    /* 创建init() */
    let init = function () {
        
        /* 调用setFontSize()和bindEvent() */
        setFontSize();
        bindEvent();
    }
    
    /* 创建bindEvent() */
    function bindEvent() {
        
        /* 全局监听resize事件，绑定setFontSize() */
        window.addEventListener('resize', setFontSize, false)
        
        /* 给菜单按钮监听click事件，绑定showNav() */
        MenuBtn.addEventListener('click', showNav, false)
    }
    
    /* 创建setFontSize() */
    function setFontSize() {
        
        /* 获取当前屏幕宽度 */
        let cWidth = doc.documentElement.clientWidth
        
        /* 如果当前屏幕宽度小于了414就动态设置根元素字体大小 */
        if(cWidth <= 414) {
            doc.documentElement.style.fontSize = cWidth / 37.5 + 'px';
        } else {
            
            /* 如果大于了414就判断一下当前根元素字体大小是不是非62.5%，满足条件就将其设置为62.5% */
            const fontSize = doc.documentElement.style.fontSize;
            if(fontSize != '62.5%') {
                doc.documentElement.style.fontSize = '62.5%'
            }
        }
        
        /* 
            判断一下屏幕宽度是不是大于768，如果是就展示导航栏，如果不是就隐藏
            因为showNav()设置导航栏的display，如果在小于768的时候关闭导航栏后，设备恢复到大于768的大小
            display不会聪明到自己恢复显示
        */
        if(cWidth > 768){
            Nav.style.display = 'block'
        } else {
            Nav.style.display = 'none'
        }
    }
    
    /* 创建showNav() */
    function showNav() {
        
        /* 获取当前导航栏的display */
        let isShow = Nav.style.display;
        
        /* 如果是block设置为none，反之亦然 */
        if(isShow === 'block'){
            Nav.style.display = 'none'
        } else {
            Nav.style.display = 'block'
        }
    }
    
    /* 执行init() */
    init()
})(document);
