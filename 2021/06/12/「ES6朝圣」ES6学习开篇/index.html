前言

ES6，全名ECMAScript 6.0，是 ECMA（European Computer Manufactures Association）组织在 2015 年 6 月发布的标准。

ECMAScript 与 JavaScript 的关系

ECMAScript 是怎么来的？

1996 年 11 月 JS 的创造者网景（NetScape）公司将 JS 提交给 ECMA，希望 JS 能成为国际标准。次年 ECMA 发布ECMA-262文件的初版，规定了浏览器脚本语言的标准，并定名位 ECMAScript。

为什么要命名为 ECMAScript？

这一标准一开始就是为 JS 量身定制，为什么不叫 JavaScript 主要有两个原因

  • 版权问题： Java 的版权在 SUN 公司手里，根据 SUN 与 NetScape 公司的授权协议，只有 NetScape 公司能合法使用 JavaScript 这个名字。
  • 保证这门语言的中立性： 制定规范的是 ECMA 而不是 NetScape。

总而言之，前者是后者是规范，后者则是前者的实现

Babel（转码器）

简介

Babel 是一个广泛使用的 ES6 转码器，可以将 ES6 的代码转为 ES5 的代码，从而在老版本的浏览器（IE）上执行。这意味着你可以使用 ES6 编程，但不用考虑现有环境是否兼容。

const testArray = [1,2,3,4];
// 转码前，ES6支持的箭头函数
testArray.map((item) => item + 1)
// 转码后，普通函数
testArray.map(function(item) => {
    return item + 1
})

配置文件

安装 Babel

npm install --save-dev @babel/core

使用 Babel 的第一步是先配置他的配置文件.babelrc，这个文件在他的项目根目录下。

{
    "presets": [], // 转码规则
    "plugins": [], // 插件
}

官方提供了一些规则集，你可以根据需求来进行安装，下面是 react 转码规则和最新的转码规则

# 最新的转码规则
npm install --save-dev @babel/presets-env
# react的转码规则
npm install --save-dev @babel/presets-react

{
    "presets": [
        "@babel/presets-env",
        "@babel/presets-react"
    ],
    "plugins": [],
}

@babel/cli 模块

⚠ 需要先配置.babelrc 才能使用

Babel 提供了命令行的转码工具@babel/cli

安装

npm install --save-dev @babel/cli

基本用法

# 转码结果输出到标准输出
npx babel example.js

# 转码结果写入到一个文件，通过-o或者--out-file来指定输出文件
npx babel example.js -o compiled.js
npx babel example.js --out-file compiled.js

# 整个目录进行转码，通过-d或者--out-dir来指定输出文件
npx babel example.js -d lib
npx babel example.js --out-dir lib

# -s 参数生成source map文件
npx babel src -d lib -s

@babel/node 模块

⚠ 需要先配置.babelrc 才能使用

@babel/node 的 babel-node 命令提供一个支持 ES6 的 REPL 的环境，可以直接运行 ES6 的代码，并且支持 Node 的 REPL 环境所有的功能。

安装

npm install --save-dev @babel/node

执行 babel-node 命令就进入 REPL 环境了，该命令也能直接运行 ES6 脚本文件

# 进入环境
npx babel-node
# 直接执行脚本文件
npx babel-node ES6.js

@babel/register 模块

⚠ 需要先配置.babelrc 才能使用

⚠@babel/register 模块只会对 require 命令加载的文件进行转码，不会对当前文件进行转码

⚠ 转码过程是实时进行的，w’w 只是适用于开发环境

@babel/register 模块给 require 命令加上了一个 Hook，每当 require 加载后缀为 js，jsx，es，es6 的文件，会通过调起 Hook 先用 Babel 进行转码。

安装

npm install --save-dev @babel/register

引用

// 必须先加载@babel/register模块
require("@babel/register");
require("./es6.js");

polyfill（垫片）

polyfill 我理解还是一个概念性的东西，你可以理解为它是在给未能实现的东西进行一个兜底，让其能正常运行

Babel 默认只转码 JS 的句法，而不会转码新的 API，Babel 默认不转码的 API 有很多，具体可以查看babel-plugin-transform-runtime模块的definition.js文件。

举个例子，Array 对象在 ES6 上新了.from方法，Babel 是不会对这个方法进行转码。如果想要这个方法有效，需要core-js与regenerator-runtime这两个模块来为当前环境提供一个垫片。

安装

npm install --save-dev core-js regenerator-runtime

用 import 或者 require 引入模块，配置脚本

import 'core-js';
import 'regenerator-runtime/runtime';

require('core-js');
require('regenerator-runtime/runtime);

总结

这里简单的讲了一下 ES6 的由来，阮一峰大佬的书里有更详细的介绍，比如 ECMAScript 的具体历史，语法提案的流程等等。然后 Babel 和 ployfill 这俩前端黑话级别的东西，目前我的开发经历中是几乎没直接使用过，这俩对我来说就相当于一个知识储备。
