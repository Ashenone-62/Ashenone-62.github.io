JS诞生之路

JS的出生

背景：1995年前，是绝大多数人网速还停留在28.8kbit/s的时代，是表单校验只能由服务器完成的时代，是会出现填写完一张表单等待30秒后因关键字段没有填写而重填的时代！如果这些东西换到今天你会不会血压拉满？会不会捶胸顿足？会不会心脏直接漏跳半拍！？

诞生：当时的互联网大头Netscape（网景）公司决定开发一门语言在客户端完成一些简单的验证任务。在那个年代，数据能在发送到服务器之前，在客户端阶段完成校验工作的壮举绝对令人兴奋。当时就职于Netscape的Brendan Eich（布兰登-艾奇）计划为1995年2月发布的Netscape Navigtor 2(Netscape的浏览器)开发一款脚本语言，最初定名为LiveScript。再后来，为了赶在发布日期前完成任务，Netscape和Sun（开发Java的公司，后被Oracle收购）成立了一个开发联盟，并在发布前夕蹭了一波Java的热度（当时的Java可谓是如日中天，属于顶流级别），将LiveScript改名为JavaScript。

微软暗中发展，拿里程碑助攻

JavaScript1.0的发布获得了巨大成功，Netscape随即在自家浏览器Netscape Navigator 3中发布了JavaScript1.1，并将自己定位为行业领袖型公司。微软一看，不干了，我好歹也是互联网大佬啊，我面子不要啦！在自家浏览器投入了更多资源，在Netscape Navigator 3发布不久后发布了Internet Explore 3，并在其中加入了JScript（本质上是JavaScript的实现，两者没有太多区别；命名上为了避免撞名JavaScript引发版权纠纷，因为当时Netscape是付了钱给Sun拿到了Java名字的使用权的）。

微软正式进军Web浏览器领域这一动作，标志着JavaScript作为一门开发语言前进了里程碑式的一步。

JS的事业编

两大巨头在JS领域的互相欧拉欧拉让市面上出现了两个版本的JS，即Netscape Navigator的JavaScript和I Internal Explorer的JScript，随着业界的担心日益加剧，JavaScript的标准化问题被提上了日程。两种版本的JS会引发啥问题呢？必然是公司为了牟利搞垄断，你用其他版本的JS在我的浏览器就跑不起来，这对一门开发语言来说是非常可怕的。

1997年，JavaScript1.1作为蓝本提交给了欧洲计算机制造商协会（European Computer Manufacturers Association，简称ECMA），由该协会的39号技术委员会负责标准化一种通用的、跨平台的、供应商中立的脚本语言的语义和语法。经过数月的努力，这个团队拿出了他们的成果ECMA-262，次年国际标准化组织和国际电工委员会（International Organization For Standardization and International Electrotechnical Commission，简称ISO/IEC）采用了ECMAScript作为标准，即ISO/IEC-16262。

此后，浏览器开发商致力于将ECMAScript作为各自JavaScript实现的基础，取得了不同程度上的成功。

ECMAScript

JavaScript是由ECMAScript、DOM、BOM组成的，接下来会简单地聊聊他们的故事。

通常谈到JS我们最先联想到的东西就是脚本、交互、浏览器。ECMAScript和Web浏览器并没有任何依赖关系，事实上ECMAScript并不包含输入和输出的定义，它只定义了JavaScript这门语言的基础，在此基础之上可以构建更完善的脚本语言。硬要说Web浏览器和ECMAScript有什么关系的话，Web浏览器上ECMAScript实现可能的宿主环境之一（宿主环境会提供基本的实现和拓展，以便语言和环境之间对接交互），Node、Adobe Flash这些是我们常见的宿主环境。

ECMAScript都规定了哪些内容呢？大致说来，它规定下面列表里的东西：

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 对象

简单来说，ECMAScrip就是对实现自身（ECMA-262）各方面的语言描述，而JavaScript实现了ECMAScript，同理Node、Adobe Flash也实现了ECMAScript。

DOM

文档对象模型（Document Object Model，简称DOM），是针对XML但经过拓展用于HMTL的应用程序编程接口（Application Programming Interface，简称API）。DOM把整个页面映射成一个多层节点结构，这些节点又包含着不同类型的数据。

<html>
  <head>
    <title>我是标题</title>
  </head>
  <body>
    <p>我是内容</p>
  </body>
</html>

上面这份HTML代码就会被映射为下面这个树形数据结构。通过DOM创建的这个树形图，开发者获得了控制页面内容和结构的主动权，能轻松自如地执行增、删、改操作。

# |html
# |- head
# |- |- title
# |- |- |- 我是标题
# |- body
# |- |- p
# |- |- |- 我是内容

DOM的事业编

随着Netscape Navigator 4和Internet Explore 4的发布，两种浏览器都支持了不同形式的Dynamic HTML（简称DHTML）。在此基础上，开发者不用刷新页面就能修改HTML的内容和外观。但是Microsoft和Netscape在DHTML方面各持己见，这意味着过去那个只编写一个HTML就能在不同浏览器上访问的时代已经结束了，DOM的标准化的问题被放在的台面上。负责Web通信标准的万维网联盟（World Wide Web Consortium，简称W3C）揽下了这个活儿。

DOM从1到3

到目前为止DOM总共有三个级别，DOM1级、DOM2级、DOM3级。在网上冲浪的时候，你有可能会看见DOM0级的字眼，事实上DOM0级是不存在的，DOM0级一般泛指DOM历史坐标中的一个时间点，即Netscape Navigator 4和Internet Explore 4最初支持的那版DHTML。

DOM1级：DOM1级于1998年10月称为W3C推荐标准，由DOM核心（DOM Core）和DOM HTML两部分组成。DOM核心规定了如何去映射基于XML的文档结构，简化了对文档中任意部分的访问和操作，而DOM HTML则在DOM核心的基础上加以拓展，添加了针对HTML的对象和方法。

DOM2级：DOM2级在原有DOM基础上拓展了鼠标和用户界面的事件，遍历DOM的方法等细分模块，而且通过对象接口增加了对CSS的支持。

DOM3级：DOM3级进一步拓展了DOM，引入了统一的加载和保存文档的方式，新增验证文档的方法。DOM3级也对DOM核心做了拓展，开始支持XML1.0规范，覆盖了XML Infoset，XPath和XML Base。

浏览器与DOM的那些事

冷知识，DOM标准出现一段时间后，浏览器才开始实现它。

Microsoft打算在IE 5中实现DOM，但在IE 5.5的时候才算真正支持DOM1级，并且在6，7版本没有更新任何和DOM相关的内容，在IE 8的时候才修复了之前DOM相关的BUG。

Netscape则被AOL收购，一部分资源被注入一个非盈利基金会——Mozilla基金会，在Netscape 6（Mozilla 0.6.0）的时候开始支持DOM。Netscape 7之后，Mozilla基金会将开发重心转向到了今天我们熟知的Firefox上面，Mozilla的目标是打造一款构建与标准100%兼容的浏览器，他们的努力也获得了回报。Firefox 3完全支持了DOM1级，几乎支持了DOM2级，甚至还支持了DOM3级的一部分功能。

BOM

DOM是文档对象模型，大致猜一猜就能知道BOM是啥东西了，BOM代表的是浏览器对象模型（Browser Object Model）。从根本上讲，BOM只处理浏览器的窗口和框架，开发者可以使用BOM来控制浏览器显示页面以外的地方，但人们习惯性的将所有针对浏览器的JavaScript拓展也算作BOM的一部分，这些拓展包括不限于下面这些：

  • 弹出新浏览器的窗口
  • 移动、缩放、关闭浏览器窗口
  • 提供浏览器详细信息的navigator对象
  • 提供加载页面详细信息的location对象
  • 提供用户显示器分辨率详细信息的screen对象
  • 对cookie的支持
  • 自定义对象，如XMLHttpRequest、IE的ActionXObject

BOM真正神奇的地方在于，它是JavaScript实现的一部分，但是没有相关标准。由于没有相关标准，因此每个浏览器都有自己的实现，对于一些约定俗成的标准，如window和navigator对象，不同的浏览器也往里面封装一些自己的属性和方法。这个问题在HTML5得到了解决，H5致力于很多BOM功能写入正式规范，BOM的高兼容性未来可期。

小结

  • JavaScript是一种专门为网页交互而设计的脚本语言，由ECMAScript、DOM、BOM三部分组成。
  • ECMAScript是对JavaScript的语言描述，JavaScript实现了ECMAScript。
  • DOM，文档对象模型，提供了访问和操作网页的方法和接口。
  • BOM，浏览器对象模型，提供了与浏览器交互的方法和接口。
