JS 诞生之路

JS 的出生

背景：1995 年前，是绝大多数人网速还停留在 28.8kbit/s 的时代，是表单校验只能由服务器完成的时代，是会出现填写完一张表单等待 30 秒后因关键字段没有填写而重填的时代！如果这些东西换到今天你会不会血压拉满？会不会捶胸顿足？会不会心脏直接漏跳半拍！？

诞生：当时的互联网大头 Netscape（网景）公司决定开发一门语言在客户端完成一些简单的验证任务。在那个年代，数据能在发送到服务器之前，在客户端阶段完成校验工作的壮举绝对令人兴奋。当时就职于 Netscape 的 Brendan Eich（布兰登-艾奇）计划为 1995 年 2 月发布的Netscape Navigtor 2(Netscape的浏览器)开发一款脚本语言，最初定名为 LiveScript。再后来，为了赶在发布日期前完成任务，Netscape 和 Sun（开发 Java 的公司，后被 Oracle 收购）成立了一个开发联盟，并在发布前夕蹭了一波 Java 的热度（当时的 Java 可谓是如日中天，属于顶流级别），将 LiveScript 改名为 JavaScript。

微软暗中发展，拿里程碑助攻

JavaScript1.0 的发布获得了巨大成功，Netscape 随即在自家浏览器 Netscape Navigator 3 中发布了 JavaScript1.1，并将自己定位为行业领袖型公司。微软一看，不干了，我好歹也是互联网大佬啊，我面子不要啦！在自家浏览器投入了更多资源，在 Netscape Navigator 3 发布不久后发布了 Internet Explore 3，并在其中加入了 JScript（本质上是 JavaScript 的实现，两者没有太多区别；命名上为了避免撞名 JavaScript 引发版权纠纷，因为当时 Netscape 是付了钱给 Sun 拿到了 Java 名字的使用权的）。

微软正式进军 Web 浏览器领域这一动作，标志着 JavaScript 作为一门开发语言前进了里程碑式的一步。

JS 的事业编

两大巨头在 JS 领域的互相欧拉欧拉让市面上出现了两个版本的 JS，即 Netscape Navigator 的 JavaScript 和 I Internal Explorer 的 JScript，随着业界的担心日益加剧，JavaScript 的标准化问题被提上了日程。两种版本的 JS 会引发啥问题呢？必然是公司为了牟利搞垄断，你用其他版本的 JS 在我的浏览器就跑不起来，这对一门开发语言来说是非常可怕的。

1997 年，JavaScript1.1 作为蓝本提交给了欧洲计算机制造商协会（European Computer Manufacturers Association，简称 ECMA），由该协会的 39 号技术委员会负责标准化一种通用的、跨平台的、供应商中立的脚本语言的语义和语法。经过数月的努力，这个团队拿出了他们的成果 ECMA-262，次年国际标准化组织和国际电工委员会（International Organization For Standardization and International Electrotechnical Commission，简称 ISO/IEC）采用了 ECMAScript 作为标准，即 ISO/IEC-16262。

此后，浏览器开发商致力于将 ECMAScript 作为各自 JavaScript 实现的基础，取得了不同程度上的成功。

ECMAScript

JavaScript 是由 ECMAScript、DOM、BOM 组成的，接下来会简单地聊聊他们的故事。

通常谈到 JS 我们最先联想到的东西就是脚本、交互、浏览器。ECMAScript 和 Web 浏览器并没有任何依赖关系，事实上 ECMAScript 并不包含输入和输出的定义，它只定义了 JavaScript 这门语言的基础，在此基础之上可以构建更完善的脚本语言。硬要说 Web 浏览器和 ECMAScript 有什么关系的话，Web 浏览器上 ECMAScript 实现可能的宿主环境之一（宿主环境会提供基本的实现和拓展，以便语言和环境之间对接交互），Node、Adobe Flash 这些是我们常见的宿主环境。

ECMAScript 都规定了哪些内容呢？大致说来，它规定下面列表里的东西：

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 对象

简单来说，ECMAScrip 就是对实现自身（ECMA-262）各方面的语言描述，而 JavaScript 实现了 ECMAScript，同理 Node、Adobe Flash 也实现了 ECMAScript。

DOM

文档对象模型（Document Object Model，简称 DOM），是针对 XML 但经过拓展用于 HMTL 的应用程序编程接口（Application Programming Interface，简称 API）。DOM 把整个页面映射成一个多层节点结构，这些节点又包含着不同类型的数据。

<html>
  <head>
    <title>我是标题</title>
  </head>
  <body>
    <p>我是内容</p>
  </body>
</html>

上面这份 HTML 代码就会被映射为下面这个树形数据结构。通过 DOM 创建的这个树形图，开发者获得了控制页面内容和结构的主动权，能轻松自如地执行增、删、改操作。

# |html
# |- head
# |- |- title
# |- |- |- 我是标题
# |- body
# |- |- p
# |- |- |- 我是内容

DOM 的事业编

随着 Netscape Navigator 4 和 Internet Explore 4 的发布，两种浏览器都支持了不同形式的 Dynamic HTML（简称 DHTML）。在此基础上，开发者不用刷新页面就能修改 HTML 的内容和外观。但是 Microsoft 和 Netscape 在 DHTML 方面各持己见，这意味着过去那个只编写一个 HTML 就能在不同浏览器上访问的时代已经结束了，DOM 的标准化的问题被放在的台面上。负责 Web 通信标准的万维网联盟（World Wide Web Consortium，简称 W3C）揽下了这个活儿。

DOM 从 1 到 3

到目前为止 DOM 总共有三个级别，DOM1 级、DOM2 级、DOM3 级。在网上冲浪的时候，你有可能会看见 DOM0 级的字眼，事实上 DOM0 级是不存在的，DOM0 级一般泛指 DOM 历史坐标中的一个时间点，即 Netscape Navigator 4 和 Internet Explore 4 最初支持的那版 DHTML。

DOM1 级：DOM1 级于 1998 年 10 月称为 W3C 推荐标准，由 DOM 核心（DOM Core）和 DOM HTML 两部分组成。DOM 核心规定了如何去映射基于 XML 的文档结构，简化了对文档中任意部分的访问和操作，而 DOM HTML 则在 DOM 核心的基础上加以拓展，添加了针对 HTML 的对象和方法。

DOM2 级：DOM2 级在原有 DOM 基础上拓展了鼠标和用户界面的事件，遍历 DOM 的方法等细分模块，而且通过对象接口增加了对 CSS 的支持。

DOM3 级：DOM3 级进一步拓展了 DOM，引入了统一的加载和保存文档的方式，新增验证文档的方法。DOM3 级也对 DOM 核心做了拓展，开始支持 XML1.0 规范，覆盖了 XML Infoset，XPath 和 XML Base。

浏览器与 DOM 的那些事

冷知识，DOM 标准出现一段时间后，浏览器才开始实现它。

Microsoft 打算在 IE 5 中实现 DOM，但在 IE 5.5 的时候才算真正支持 DOM1 级，并且在 6，7 版本没有更新任何和 DOM 相关的内容，在 IE 8 的时候才修复了之前 DOM 相关的 BUG。

Netscape 则被 AOL 收购，一部分资源被注入一个非盈利基金会——Mozilla 基金会，在 Netscape 6（Mozilla 0.6.0）的时候开始支持 DOM。Netscape 7 之后，Mozilla 基金会将开发重心转向到了今天我们熟知的 Firefox 上面，Mozilla 的目标是打造一款构建与标准 100%兼容的浏览器，他们的努力也获得了回报。Firefox 3 完全支持了 DOM1 级，几乎支持了 DOM2 级，甚至还支持了 DOM3 级的一部分功能。

BOM

DOM 是文档对象模型，大致猜一猜就能知道 BOM 是啥东西了，BOM 代表的是浏览器对象模型（Browser Object Model）。从根本上讲，BOM 只处理浏览器的窗口和框架，开发者可以使用 BOM 来控制浏览器显示页面以外的地方，但人们习惯性的将所有针对浏览器的 JavaScript 拓展也算作 BOM 的一部分，这些拓展包括不限于下面这些：

  • 弹出新浏览器的窗口
  • 移动、缩放、关闭浏览器窗口
  • 提供浏览器详细信息的 navigator 对象
  • 提供加载页面详细信息的 location 对象
  • 提供用户显示器分辨率详细信息的 screen 对象
  • 对 cookie 的支持
  • 自定义对象，如 XMLHttpRequest、IE 的 ActionXObject

BOM 真正神奇的地方在于，它是 JavaScript 实现的一部分，但是没有相关标准。由于没有相关标准，因此每个浏览器都有自己的实现，对于一些约定俗成的标准，如 window 和 navigator 对象，不同的浏览器也往里面封装一些自己的属性和方法。这个问题在 HTML5 得到了解决，H5 致力于很多 BOM 功能写入正式规范，BOM 的高兼容性未来可期。

小结

  • JavaScript 是一种专门为网页交互而设计的脚本语言，由 ECMAScript、DOM、BOM 三部分组成。
  • ECMAScript 是对 JavaScript 的语言描述，JavaScript 实现了 ECMAScript。
  • DOM，文档对象模型，提供了访问和操作网页的方法和接口。
  • BOM，浏览器对象模型，提供了与浏览器交互的方法和接口。
