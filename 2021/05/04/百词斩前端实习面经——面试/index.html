面试答案

Vue

  • Vue的双向绑定是怎么实现的？
    • 答案：基于数据劫持和订阅发布者模式来实现的。Observe，用来遍历数据对象的数据然后通过Object.defineProperty来劫持getter和setter，当数据有变动的时候就去发布通知。Compile，用来解析模板，将模板中的变量替换成数据，同时绑定更新函数，设置监听回调的订阅者，一旦收到model层的数据变更通知，就更新视图。Watcher，是打通Observe和Compile的桥梁，会在自身实例化的时候添加进属性订阅器里，然后它必须得有一个update函数，然后收到数据变动的通知，就调用update然后触发Compile的监听回调。最后MVVM整合三者，达到Model数据变动更新视图，视图和用户交互更新引起Model的数据更新。
  • Vue中Babel是如何编译的？
    • 答案： 了解AST

React

  • 会哪些Hooks
    • 答案：根据自身情况作答
  • useState是怎么工作的
    • 答案：useState原理
  • useMemo和useCallback有什么差别
    • 答案：useMemo返回函数的结果，useCallback返回函数，都是用来避免不必要的渲染的，具体可以去看下我的React Hooks——useMemo
  • 使用useState修改state的时候会马上更新吗
    • 会——>×
    • 不会——>√——>为什么不会马上更新
      • 答案：因为在react合成事件中改变状态是异步的，出于减少render次数，react会收集所有状态变更，然后比对优化，最后做一次变更
  • useEffect的依赖可以写函数吗，不写全依赖能跑起来吗
    • 答案：可以，能跑但是不安全，这里就以函数举例，你的在useEffect调用的函数可能使用了外界的值，所以依赖一定要写全
  • useEffect中返回一个函数可以用来做什么
    • 答案：一般是用来执行销毁/卸载的时候的操作，用来解绑事件监听或者定时器

JS

  • ES6有哪些新特性
    • 答案：ES6新特性总结
  • Promise是怎么工作的
    • 答案：Promise与Async总结
  • 传入Promise的函数中通常有resolve和reject两个方法作为参数，这两个方法的返回值是什么
    • 答案：resolve直接返回一个fullfilled状态的 Promise 对象，reject直接返回一个rejected状态的 Promise 对象
  • 使用setTimeout()或者setInterval()，当其中代码执行完了，所涉及的数据会不会马上更新
    • 会——>×
    • 不会——>那JS是怎么处理他们的呢
      • 答案：JS处理异步操作时候，会将其挂起然后继续执行后面的内容，挂起的部分执行完了会返回一个回调，根据回调的类型把他加入对应的宏队列队列或者微任务队列，然后本次所有内容执行完了再执行微任务队列，然后再执行宏任务队列。
  • JS里有哪些微任务和宏任务
    • 答案：微任务有promise的回调，node的nextTick，监听DOM的mutationObserve，宏任务script加载执行，ui渲染，io交互，延时函数setTimeout，setInterval

网络

  • OSI模型有哪些层
    • 答案： 从上往下，应用层，表示层，会话层，传输层，网络层，数据链路层，物理层
  • 应用层有哪些协议
    • 答案：HTTP，FTP，SMTP，DNS，Telnet等等
  • HTTP和HTTPS有什么差别
    • 答案：HTTP->TCP->IP->数据链路层->…；HTTP->SSL/TLS->TCP->IP->数据链路层->…
  • HTTP请求是怎么传输数据的，或者HTTP请求要传输的数据放在哪
    • 答案：GET请求的参数在HTTP中是通过URL传递的，而POST请求的数据是通过requestbody传递的
  • form-data和json有什么区别
    • 答案：传递参数格式为form-data与json的区别，以及ajax、axios、fetch请求的方式
  • TCP协议怎么怎么进行数据传输的
    • 答案：三次握手，传输数据，四次挥手

css

  • display有哪些值
    • 答案：none，block，inline，inline-block，flex，table，list-item，inherit
  • position有哪些值
    • 答案：static，relative，absolute，fixed，inherit
  • display:none和visibility:hidden有什么差别
    • 答案：display: none 不会出现在渲染树里，渲染的时候不会占据任何空间；visibility:hidden 会出现在渲染树立，渲染的时候会占据空间，只是不显示。display是非继承属性，子孙节点也会消失，修改子孙节点属性也不会显示；visibility是可继承属性，子孙节点也会消失是因为继承了hidden属性，将其修改为visible就可以显示。修改通常元素中的display会造成当前元素重排，修改visibility只会造成当前元素重绘
  • 详细说一下flex，如何使用flex来实现三栏布局
    • 答案：先详细说一下flex是个啥，有啥属性，然后实现三栏布局就是固定左右盒子缩放比例，中间flex:1占满剩余空间
  • 详细说一下BFC
    • 答案：块格式化上下文（Block Formatting Context，BFC）是Web页面的可视化CSS渲染的一部分，是布局过程中生成块级盒子的区域，也是浮动元素与其他元素的交互限定区域。BFC是一个独立的布局环境，可以理解为一个容器，在这个容器中按照一定规则进行物品摆放，并且不会影响其它环境中的物品。如果一个元素符合触发BFC的条件，则BFC中的元素布局不受外部影响。创建BFC的条件：根元素：body；元素设置浮动：float 除 none 以外的值；元素设置绝对定位：position (absolute、fixed)；display 值为：inline-block、table-cell、table-caption、flex等；overflow 值为：hidden、auto、scroll。BFC的作用：解决margin的重叠问题；解决高度塌陷的问题；创建自适应两栏布局。
  • 实现一个元素向右移动有哪些方式
    • 答案：margin-left，left，transform:translate(x,y)
  • margin-left和transform:translate(x,y)脱离文档流了吗
    • 答案：没有，前者控制外边距，后者相当于相对定位
