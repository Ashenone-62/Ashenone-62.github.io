let

let 的用法就像 ES5 中的 var 一样，但是不允许在一个作用域内重复声明

var ZS = "ZhangSan";
let LS = "LiSi";

var ZS = "JiaZhangSan"; // 覆盖掉ZhangSan
let LS = "JiaLiSi"; // SyntaxError

暂时性死区

使用 let 声明的变量不会出现变量提升。在 ES5 中用 var 声明变量，在进入环境的时候变量就存在了（变量提升），值为 undefined，而使用 let 声明的变量在声明前是不存在的，在其声明前使用会报错

console.log(LS); // ReferenceError
let LS = "LiSi";

ES6 中规定了，只要块级作用域中存在 let 命令，声明的变量就会绑定这个块级作用域，从一开始就形成封闭作用域，不受外界影响，在声明之前使用就会报错。let 命令声明之前的那段时间称为暂时性死区

LS = "LiSi"; // 死区开始↓
console.log(LS); // ReferenceError

let LS; // 死区结束↑
console.log(LS); // undefined

LS = "LiSi";
console.log(LS); // LiSi

还有一种情况是变量没被完全声明的时候去使用它也会报错

let x = x; // 未完成，ReferenceError

let x;
x = x; // 完成，undefined

块级作用域

在之前只有全局作用域和函数作用域两个概念，ES6（✨let，const✨）的出现带来了块级作用域的概念，几乎取代了原先的立即执行函数的写法。使用 let 声明的变量只在 let 命令所在的代码块内有效。

for (var i = 0; i < 10; i++) {}

for (let j = 0; j < 10; j++) {}

console.log(i); // 10
console.log(j); // ReferenceError

🚨ES6 的块级作用域必须要有大括号🚨

if(true) let test = 0; // SyntaxError，被当作当前作用域的顶层
if(true) { let test = 0; }

if(true) function func() {}; // 函数在严格模式下只能声明在当前作用域的顶层
if(true) { function func() {}; }

可以避免作用域之间变量污染

块级作用域可以避免内层变量污染外层变量，这种问题在 ES5 经常出现

/* ES5 */
{
  var test = 1;
  {
    var test = 2;
    console.log("inner", test); // 2
  }
  console.log("outer", test); // 2
}

/* ES6 */
{
  let test = 1;
  {
    let test = 2;
    console.log("inner", test); // 2
  }
  console.log("outer", test); // 1
}

还有一种情况就是开始提到的 for 循环，用 var 会导致 i 泄漏到全局环境

for (var i = 0; i < 10; i++) {}

for (let j = 0; j < 10; j++) {}

console.log(i); // 10，泄漏成了全局变量
console.log(j); // ReferenceError，只能存留在循环的作用域中，循环结束被销毁

需要避免在块级作用域中声明函数

ES5 和 ES6 处理块级作用域中的函数的行为差别极大。ES5 中在块级作用域声明函数是非法的，但是浏览器为了兼容，还是支持在块级作用域中声明函数。ES6 中在块级作用域中声明函数和 let 声明变量类似，没有声明就不能使用。

// ES5，下面两种情况理论上是非法的，实际上可以运行不会报错
if (true) {
  function fun() {}
}

try {
  function f() {}
} catch (e) {}

下面以一个例子来说明 ES5 和 ES6 处理块级作用域中的声明函数的差别

/* ES5 */

// 最终会打印出I'm inner，自执行函数中的f()被提升到了首部
function f() {
  console.log("I'm outer");
}
(function () {
  if (false) {
    function f() {
      console.log("I'm inner");
    }
  }
  f();
})();

/* ES6 */

// TypeError，因为false代码没有走到声明f()，所以自执行函数中的f()没被声明
function f() {
  console.log("I'm outer");
}
(function () {
  if (false) {
    function f() {
      console.log("I'm inner");
    }
  }
  f();
})();

上面的例子说明 ES5->ES6 的处理块级作用域中声明函数的方式是完全不同的，因此 ES6 的规范提到了下面三个点来解决兼容问题

🚨仅限支持 ES6 的浏览器支持，不支持 ES6 的浏览器依旧按照类似 let 的方式处理🚨

  • 允许在块级作用域内声明函数
  • 函数声明类似于 var，即会提升到全局作用域或函数作用域的头部
  • 函数声明还会提升到所在的块级作用域的头部

const

const 声明的是一个只读常量，一旦声明就不可变，也就意味着一旦声明就必须初始化不能留着过年再赋值。const 和 let 一样拥有块级作用域、暂时性死区和不可重复声明等特性。const 是让变量指向的内存地址中保存的数据不得变动来实现保持数据不变，如果保存的数据是基础数据类型，那么就相当于保存了一个常量，但是如果保存的是一个对象，那它并不是安全的，因为内存地址中保存的实际上是指向堆内存的一个指针，对象本身依旧是可写的状态。

const user = {};

// 为 user 添加一个属性，可以成功
user.name = "ZhangSan";
console.log(user.name); // ZhangSan

// 将 user 指向另一个对象，就会报错
foo = {}; // TypeError: "user" is read-only

如果想要对象不可变，需要使用 Object.freeze()来冻结，这个方法会返回一个不可变对象。

🚨 需要注意的是即使是 Object.freeze()也只是浅层冻结，如果数组或者对象内部还有引用其他数组和对象，需要通过遍历来把每个属性都冻上 🚨

var constantize = (obj) => {
  Object.freeze(obj);
  Object.keys(obj).forEach((key, i) => {
    if (typeof obj[key] === "object") {
      constantize(obj[key]);
    }
  });
};

ES6 的变量声明方式

总共有六种方式

  • var
  • let
  • const
  • function
  • import
  • class

顶层对象

顶层对象（全局对象）————浏览器里指 window 对象，Node 中指 global 对象。在 ES6 之前顶层对象的属性和全局变量是挂钩的，由于 window 对象在浏览器中还作为一个实例（封装了一些浏览器的接口和方法），因此照成了相当多的麻烦————编译阶段不能发现错误；开发者不小心创建全局变量（顶层对象属性）；顶层对象属性可读写随处可见等等。

ES6 中规定 var 和 function 声明的变量和函数依旧属于顶层对象属性，由 const，let，class 声明的全局变量则不属于顶层对象属性，进一步实现全局变量和顶层对象属性的解耦。

var a = 1;
window.a; // 1

let b = 1;
window.b; // undefined

globalThis 对象

上面提到的顶层对象在各个环境中是不统一的，globalThis 提供了一个标准的方式来获取不同环境下的全局 this 对象（也就是全局对象自身）

  • 在浏览器中顶层对象是 window，但 Node 和 Web Worker 没有
  • 在浏览器和 Web Worker 中 self 指向顶层对象，但 Node 没有
  • 在 Node 中，顶层对象是 global，但浏览器和 Web Worker 没有

this 的出现一定程度上解决了这个问题，但也有局限性

this 详解

  • 全局环境下 this 指向顶层对象，在模块中就出现了分歧，Node 模块中 this 指向当前模块，ES6 中 this 则返回 undefined。
  • 当函数不作为方法调用的时候，this 指向的是顶层对象，严格模式下则返回 undefined
  • 不管什么模式 new Function(‘return this’)() 都会返回顶层对象，但是如果开发者使用了内容安全策略（CSP），eval 和 new Function 都不能使用
/* this的解决方案 */

// 方法一
typeof window !== "undefined"
  ? window
  : typeof process === "object" &&
    typeof require === "function" &&
    typeof global === "object"
  ? global
  : this;

// 方法二
var getGlobal = function () {
  if (typeof self !== "undefined") {
    return self;
  }
  if (typeof window !== "undefined") {
    return window;
  }
  if (typeof global !== "undefined") {
    return global;
  }
  throw new Error("unable to locate global object");
};

有了 globalThis 之后，各个环境都可以通过 globalThis 来获取顶层对象

/* 使用用例 */
if (typeof globalThis.setTimeout !== "function") {
  console.log("此环境中没有 setTimeout 方法！");
}
