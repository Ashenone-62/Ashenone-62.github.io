受控组件与非受控组件浅出

前言

在实习期间参与的第一个版本迭代中，我分到的需求有编辑相关的功能，我思路几乎都是通过ref去获取表单控件的值再去修改（非受控组件），中间碰了很多壁 😭 遂来研究受控和非受控组件。

概念

在React中，一般推荐使用受控组件，因为受控组件的状态和方法一般都是由顶层组件来维护，拓展性显而易见的好。当然也不是说受控组件好就要全部使用受控组件，比如file类型表单控件的值只能由用户传入，因此只能是非受控组件，我们如果要将其做成受控组件，在保存它的值的时候就会报错 ⚠

非受控组件

非受控组件顾名思义就是不受开发者控制的组件。React不支持像Vue那样实现双向绑定，在React中创建一个input组件，是不受开发者控制的，用户在input键入的内容会在自身的 state 中进行维护（这个 state 是抽象意义上的，你可以理解为它就保存在input的value这个state里，你开发者不做处理就是拿不到 😊 ）

既然非受控组件不受我们控制那我们怎么去拿呢？答案是ref，我们直接去获取DOM里的value，用户传的什么我们就要什么。

import React, { Component } from "react";

export default class UnControllComponent extends Component {
  constructor(props) {
    super(props);
    this.inputRef = React.createRef();
  }
  handleSubmit = (e) => {
    console.log("我们可以获得input内的值为", this.inputRef.current.value);
    e.preventDefault();
  };
  render() {
    return (
      <form onSubmit={(e) => this.handleSubmit(e)}>
        <input defaultValue="非受控组件" ref={this.inputRef} />
        <input type="submit" value="提交" />
      </form>
    );
  }
}

受控组件

知道了非受控组件，那受控组件的概念也就明白了，那就是受开发者控制的组件。上面提到了input的例子，用户键入的内容是存在一个抽象state里（其实就是value）不受我们控制。当我们把state的值指向我们自己设置的值时，那它就被我们控制了。

/** ...省略的代码... **/

//比如我这儿有个state
constructor(props) {
    super(props)
    this.state = {
        inputValue: "受控组件"
    }
}

//将input的value值指向this.state.inputValue
render() {
    return <input value={ this.state.inputValue } />
}

/** ...省略的代码... **/

当你这样做了，就会出现一个问题————我们把value定死了，input现在是在我们的控制之下，我们给什么它就显示什么，用户没法键入任何内容 ⚠ 这就需要下面的操作了。

/** ...省略的代码... **/

constructor(props) {
    super(props)
    this.state = {
        inputValue: "受控组件"
    }
}

//当input的内容改变的时候就通过setState将用户键入的值保存在我们的状态里,state里的值又给到了组件的value
handleChange(e) {
    this.setState({
        inputValue: e.target.value
    })
}

render() {
    <input value={ this.state.inputValue } onChange={ (e) => this.handleChange(e) } />
}

/** ...省略的代码... **/

这样我们就得到了一个受控的input组件 ✨✨✨

使用场景

场景 非受控组件 受控组件

在提交的时候验证 √ √
字段实时验证 × √
存在条件禁用提交按钮 × √
强制格式化 input 内容 × √

实例

在这里实现一个简单编辑功能 🙌 需要先安装antd组件库 ⚠

  • 默认渲染一个写有受控组件的的文本（Space组件），旁边有一个编辑按钮

  • 点击编辑按钮弹出模态框，模态框内有一个Input组件，Input组件内为Space中的内容

  • 修改内容后，点击确定，关闭模态框，结果会同步到外面；点击取消，关闭模态框，内容不会有改变。

import React, { useState } from "react";
import { Space, Button, Modal, Input } from "antd";
import "antd/dist/antd.css";

const ControlledComponent = () => {
  //管理模态框外的值
  const [outerValue, setOuterValue] = useState("受控组件");
  //管理模态框内的值，如果你希望模态框内的模态框外的值同步变化可以不要这部分的代码
  const [innerValue, setInnerValue] = useState(outerValue);
  //管理模态框开关
  const [visiable, setVisiable] = useState(false);

  //当你点击模态框的确定的时候
  const handleOnOk = () => {
    //将模态框内的值设置给外界
    setOuterValue(innerValue);
    //关闭模态框
    setVisiable(false);
  };

  return (
    <>
      <Space align="center">
        {outerValue}
        {/* 点击编辑的时候打开模态框 */}
        <Button type="primary" onClick={() => setVisiable(true)}>
          编辑
        </Button>
      </Space>
      {/**
       * 点击确定的时候执行handleOnOk
       * 点击取消或者右上角的×的时候关闭模态框
       **/}
      <Modal
        title="编辑"
        visible={visiable}
        onCancel={() => setVisiable(false)}
        onOk={handleOnOk}
      >
        {/* Input组件的值改变的时候，把改变的值设置给模态框内的状态 */}
        <Input
          value={innerValue}
          onChange={(e) => setInnerValue(e.target.value)}
        />
      </Modal>
    </>
  );
};

export default ControlledComponent;
