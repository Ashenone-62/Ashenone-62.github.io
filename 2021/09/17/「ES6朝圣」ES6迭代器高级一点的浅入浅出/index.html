前言

时过境迁，C 端魔鬼迭代迎来一小段 🈳️ 闲时间，趁这会儿赶紧把 Generator 剩余的内容更了。

yield*

一般情况，在 generator 中调用另外一个 generator 需要手动遍历。ES6 中提供了一种更简洁的方式在 generator 内部执行另一个 generator。其实使用过后不难看出yield*是 for…of…的一种简写形式，这里存在一个问题，如果yield*后面跟的迭代器有 return 语句是执行不到的，需要定一个一个变量来接 return 的值，如const res = yield* gen();。

/**
 * 常规
 * */
function* gen1() {
  yield 3;
  yield 4;
}

function* gen2() {
  yield 1;
  yield 2;
  for (const g1 of gen1()) {
    console.log(g1);
  }
}

for (const g2 of gen2()) {
  console.log(g2);
}

// 1
// 2
// 3
// 4

/**
 * ES6
 * */

function* gen1() {
  yield 3;
  yield 4;
}

function* gen2() {
  yield 1;
  yield 2;
  yield* gen1();
}

for (const g2 of gen2()) {
  console.log(g2);
}

// 1
// 2
// 3
// 4

既然yield*是 for…of…的简写，又可以推出凡事带有 Symbol.iterator 接口的数据都可以使用yield*来遍历，那么对于遍历数组这种原生具备 Symbol.iterator 的数据的成员的时候就显得异常方便了。

const tree = [1, [2, 3, [4, 5], 6]];

function* getItem(tree) {
  for (const item of tree) {
    if (Array.isArray(item)) {
      yield* getItem(item);
    } else {
      yield item;
    }
  }
}

console.log(...getItem(tree));

// 1 2 3 4 5 6

yield* 中序遍历完全二叉树。前序，后续

// 🌲节点
function Tree(left, label, right) {
  this.left = left;
  this.label = label;
  this.right = right;
}
// 中序遍历函数，前序、后续作相应调整
function* order(tree) {
  if (tree) {
    yield* order(tree.left);
    yield tree.label;
    yield* order(tree.right);
  }
}
// 创建🌲函数
const createTree = (nodeList) => {
  if (nodeList.length === 1) return new Tree(null, nodeList[0], null);
  return new Tree(
    createTree(nodeList[0]),
    nodeList[1],
    createTree(nodeList[2])
  );
};
// 🌲实例
const tree = createTree([[[1], 2, [3]], 4, [[5], 6, [7]]]);

// 遍历🌲
for (const leave of order(tree)) {
  console.log(leave);
}

// 结果： 1, 2, 3, 4, 5, 6, 7

this 及其巧用

Generator 函数总是返回一个遍历器，是 Generator 函数的实例，继承了 prototype 对象上的方法。但是并不能把定义的function*{}当作构造函数，因为它总是返回 Iterator 对象，并非 this 实例。

那么要如何获取迭代器的 this 呢？

  • 定义一个空对象，然后对齐使用 call 方法绑定 generator 内部的 this
    • 下面的的代码在执行了 yield 后才会给实例设置属性 a，b，所以需要执行两次 next。
  • 把上面方法的空对象换成 generator 实例的 prototype
    • 第一种，执行的对象上 genThis，生成的对象上 target 二者事分离。这种方式则把对象统一了，我愿称其为追根溯源绑定祖宗法。
/**
 * 第一种
 * */
function* gen() {
  yield (this.a = 1);
  yield (this.b = 2);
}

console.log(gen.a, gen.b);
// undefined, undefined

const target = {};
const genThis = gen.call(target);

genThis.next();
genThis.next();

console.log(target.a, target.b);
// 1, 2

/**
 * 第二种
 * */
function* gen() {
  yield (this.a = 1);
  yield (this.b = 2);
}

console.log(gen.a, gen.b);
// undefined, undefined

const genThis = gen.call(gen.prototype);

genThis.next();
genThis.next();

console.log(genThis.a, genThis.b);
// 1, 2

上面第二种方法稍微改造一下成构造函数，甚至能用 new 命令

function* gen() {
  yield (this.a = 1);
  yield (this.b = 2);
}

console.log(gen.a, gen.b);
// undefined, undefined

function Gen() {
  return gen.call(gen.prototype);
}

const genThis = new Gen();

genThis.next();
genThis.next();

console.log(genThis.a, genThis.b);
// 1, 2

Generator 出现得意义

Generator 的出现有重大意义，省略 10w 字……

状态机

Generator，状态机，SUPER BEST MATCH！YEAH！Generator 是状态机的最佳实现结构——Generator 本身就管理了一个状态信息，即这个状态是否暂停。

下面的代码是 ES5 和 ES6 实现状态机的方式，可以看出 ES6 的写法中不用去定义保存状态的外部变量，且代码更加简洁，更符合函数式编程的思维。由于不用去管理外部变量，被非法传改的机会大大减少，提升了代码的安全性。

// ES5
var isTicking = true;

const clock = function (isTicking) {
  this.next = function () {
    if (isTicking) {
      console.log("tic");
    } else {
      console.log("toc");
    }

    isTicking = !isTicking;
  };
};

const CLOCK = new clock(isTicking);

CLOCK.next();
CLOCK.next();
CLOCK.next();

// tic toc tic

// ES6

function* Clock() {
  while (true) {
    yield "tic";
    yield "toc";
  }
}

const clock = Clock();

console.log(clock.next());
console.log(clock.next());
console.log(clock.next());

// { value: 'tic', done: false }
// { value: 'toc', done: false }
// { value: 'tic', done: false }

协程

协程是一种程序的运行方式，可以理解为协作的线程/协作的函数。可以用单线程实现，也可以用多线程实现，前者是一种特殊的子例程，后者是一种特殊的线程。

传统子例程采用堆栈先进后出的执行方式，只有全部的子程序执行完了，才会执行父程序。而协程可以并行跑多个线程，但是只有一个线程是运行的状态，其他都是暂停的状态，这些线程之间可以互相交换执行权，也就是说一个程序可以执行到一半把执行权交出去暂停执行，之后回收执行权后再恢复执行。这种并行线程且可以交换执行权的特殊线程被称为协程。

相较于普通线程，协程也拥有执行上下文和共享全局变量，但是同一时间内运行的协程只有一个，普通线程则可以有多个。普通线程执行的时候属于抢占式，完全依托于环境；协程则可以自由分配执行权。

从实际上来看，一个线程维护了一个栈，而协程是跑并行的线程，则维护了多个栈，随之而来的就是内存开销变高。也就是说协程是以内存开销提升为代价实现的多任务的并行。

😮‍💨 看完前面一大堆内容，想必肯定已经充分理解了 💪

Generator 如果要实现协程，可以将需要互相协助的任务封装进 Generator 函数里，通过 yield 来分配执行权。但是！Generator 是 ES6 对协程的不完全实现，是半协程！因为真正的协程，任何函数都可以重启暂停的协程，但是 Generator 函数如果要重启，只能由函数调用者来分配执行权。

上下文

JS 运行的时候会产生一个上下文包含了所有的变量和对象，即运行环境。在执行函数或块级作用域的时候，会产生一新的上下文来作为当前的上下文，最终形成一个堆栈的结构。在执行的时候遵循先进后出的原则，后压入的先执行，直到所有代码执行完成后堆栈清空。

而对 Generator 来说，一遇到 yield 命令就会暂时退出它的堆栈，但是堆栈并不消失，里面所有的内容都处于一个冻结的状态，在碰见的 next 的时候则重新回到堆栈中，将 generator 的冻结状态的上下文变为当前的上下文继续执行。

Generator 应用场景

异步问题同步式表达

function* load() {
  while (true) {
    modalopen();
    yield getModalData();
    yield modalClose();
  }
}

const modalLoad = load();

// 打开modal并且加载数据
modalLoad.next();
// 关闭modal
modalLoad.next();

控制流管理

如果一个操作跟老太婆的裹脚布一样又臭又长，要层层回调，直接进到十八层回调地狱。

const step = funciton(val1) {
  return function step2(val2) {
      return function step3(val3) {
        // ....
    }
  }
};

使用 Promise 可以简化一下代码。

const PromiseStep = Promise.resolve(() => step1)
  .then(() => step2)
  .then(() => step3)
  .then(() => step4);
// ....;

上面的代码，看起来已经很简洁了 8⃣️，但是！但是啊，我们希望是对流程的一个管理，Promise 一旦执行后就无法停止，只有执行结束或者报错才会停止。Generator 能进一步满足我们的需求。

function* genSteps(value1) {
  try {
    const value2 = yield step1(value1);
    const value3 = yield step1(value2);
    const value4 = yield step1(value3);
    // ...
  } catch (e) {
    console.log(e);
  }
}

// 任务维度
const tasks = [func1, func2, func3];
function* doTask() {
  for (const task of tasks) {
    yield task();
  }
}

// 任务集维度
const tasks1 = [func1, func2, func3];
const tasks2 = [func4, func5, func6];
const tasks3 = [func7, func8, func9];
const jobs = [tasks1, tasks2, tasks3];

function* doTask(jobs) {
  for (const job of jobs) {
    yield* doTask(job);
  }
}

部署 Symbol.Iterator 接口

Generator 可以给任意对象部署 Symbol.Iterator 接口，及其方便 😏

function* setObjIterator(obj) {
  const keys = Object.keys(obj);
  for (const index in keys) {
    const realKey = keys[index];
    yield [realKey, obj[realKey]];
  }
}

const objIterator = setObjIterator({
  username: "张三",
  age: 12,
});

for (const [key, value] of objIterator) {
  console.log(key, value);
}

// username 张三
// age 12
