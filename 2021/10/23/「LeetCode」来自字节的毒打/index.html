面试官：这个 xxx 你了解吗？

我：嗯了解，这个 xxx 是巴拉巴拉巴拉巴拉，我们会巴拉巴拉巴拉去用这个 xxx…(省略一堆东西)

面试官：那你能说一下 xxx 里的 xxxx 和 yyyy 有什么区别吗？

属于是直接喜提新一届八股文大神，接下来就应了那句广为流传的名言——有人相爱，有人开车看海，有人 leetcode 第一题做不出来 😢（平时没怎么刷题今天才发现的）

题目

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。

思路

暴力破解

直接两层循环，将所有情况枚举出来进行判断是否等于 target，如果等于就直接返回两个数的下标。

hashMap

外部设置一个 map 数据结构用于保存之前的遍历过的值，每次便利一开始就去查询 map 中是否有 target 和当前遍历数的差值，如果有直接就返回两个数的下标，如果没有就将当前遍历数的值作为 key，idx 作为 value 存入表中。这样一次循环就能找出值了 👍👍。

代码

const nums = [1, 2, 3, 4];
const target = 3;

// 暴力解法
const twoSum_force = (nums, target) => {
  const len = nums.length;

  for (let i = 0; i < len - 1; i++) {
    for (let j = i + 1; j < len; j++) {
      if (nums[i] + nums[j] === target) {
        return [i, j];
      }
    }
  }
};

// hash表解法
const twoSum_hashMap = (nums, target) => {
  const hashMap = new Map();

  for (let i = 0; i < nums.length; i++) {
    if (!Number.isNaN(Number(hashMap.get(target - nums[i])))) {
      return [hashMap.get(target - nums[i]), i];
    } else {
      hashMap.set(nums[i], i);
    }
  }
};

console.log(twoSum_force(nums, target));
console.log(twoSum_hashMap(nums, target));

总结

这次字节面试虽然是透心凉心飞扬，但也带来一些启发。

首先是算法题，一开始我其实是想到了两次循环暴力解法，但是觉得太 low 了就去思考其他方法了（说得跟 hashMap 方法好难一样 🤡）。归根结底还是算法刷少了，而且思维方式停留到平时做业务的模式——因为排的事件比较足会去优先想最优解，面试应该还是得主打短时间内解决问题为主，方法虽然 low 一点但是把问题解决了，优化可以后续慢慢来。

  • 面试官：那你能讲一下日常开发中怎么选择这些方法以及为什么要选吗？
  • 我：。。。。
